# MinewESLv3KitDocumentation

This set of SDK only supports Bluetooth ESL devices produced by Minew. The SDK can help developers handle all the work between the phone and ESL, including: scanning the device, connecting the device, writing data to the device, receiving data from the device, etc.

MinewESLV3Kitcan be used in AppCompat orandroidx environment

# Preliminary work

Overall framework: MTTagBleManager is a device management class, which is always a singleton when the APP is running. TagModuleis a device instance class. This suite will generate a TagModule instance for each device, which contains device broadcast data, which will be updated as the device keeps broadcasting.

MTTagBleManager: a device management class that can scan the surrounding ESL devices and connect them, verify them, etc;

TagModule:device instance obtained during scanning, used during the scanning phase; It can be obtained when starting to connect. As an instance of the connected device, read and write operations will be used.

# Start development

sdk is divided into two stages of scanning and connection.

# Scanning

//Get management class singleton
MTTagBleManager tagBleManager = MTTagBleManager.getInstance();

//You can set the default scan time in milliseconds before you start scanning. 
//Set the scan time to be no less than 3 seconds
tagBleManager.setDefaultScanTime(10*1000);

//Clear all previous scan results
tagBleManager.clearScanResult();
//Start scanning and set the listener, the default scan is 60s
tagBleManager.getScanTagManager().startScan(this, new OnScanTagResultListener() {
    @Override
    public void onScanTagResult(List<TagModule> list) {
		//Process scan results here
    }

    @Override
    public void onStopScan(List<TagModule> list) {
		//Automatically stop scanning and return all devices acquired during the scan time
    }
});

//Or use: start scanning and set the listener, set the scan to 8 seconds, set the scan time to be not less than 3 seconds
tagBleManager.getScanTagManager().startScan(this,8000， new OnScanTagResultListener() {
    @Override
    public void onScanTagResult(List<TagInfo> list) {
		//Process scan results here
    }

    @Override
    public void onStopScan(List<TagInfo> list) {
		//Automatically stop scanning and return all devices acquired during the scan time
    }
});

# Connection

The surrounding tag device can be obtained through the scanning method, and then the specified tag can be connected. Before the connection, the key must be written before the connection method can be called. Connect by calling connect ().

tagBleManager.connect(tagModule)
1

During the connection phase, including subsequent write operations, a listener needs to be set to determine the status. Brushing can be done after success, and callback notification can also be obtained here after brushing successfully.

tagBleManager.setOnConnStateListener(new OnConnStateListener() {
    @Override
    public void onUpdateConnState(String s, ConnectionState connectionState) {
        switch (connectionState) {
            case Connecting:
               //Start to connect
                break;
            case Connect_Complete:
                // Authentication is successful, at this time, the connection is considered successful
                break;
            case BRUSH_IMAGE:
               // Called after successful drawing, the connection will be disconnected
                break;
            case Disconnect:
                //Disconnect
            default:
                break;
        }
    }
});

# Brush

After the connection is successful, you can go to brush. First you need to get the image data written into the tag. Due to the limitation of the Bluetooth data transmission length, if it exceeds the limit, it will be subcontracted and then sent in turn.

//The screen information is stored in the DeviceInfoFrame class
DeviceInfoFrame frame = (DeviceInfoFrame) tagModule.getMinewFrame(FrameType.DEVICE_INFO_FRAME);
//Remove the screen information, and then get the brush data according to the screen information
int screenIdentifier = frame.getScreenIdentifier();
String macAddress = tagModule.getMacAddress();
//The device screen is divided into double-sided screen and single-sided screen. Now it is a demonstration of double-sided screen. The //image data of side A is imageABytes, and the image data of side B is imageBBytes. The data of side A and side B are different. Set //isSameImage=false,
//If the brushing data of side A and side B are the same, setting isSameImage=true can improve the brushing efficiency
//If the device only supports single-sided cards, the A-sided screen is refreshed by default, and the imageABytes data is set, //imageBBytes=null, isSameImage=false,
//In the brushing scenario, `onWriteSuccess` will be executed first, and then the `BRUSH_IMAGE` state will be returned through //`OnConnStateListener`; the connection will be disconnected after the writing fails, and the brushing failure will be notified through //`onWriteFailed` and the connection will be disconnected.
tagBleManager.sendImageData(macAddress, imageABytes,imageBBytes,false, new OnWriteLargeDataListener() {
    
    /**
     * Update write progress
     *
     * @param index Current number written
     * @param total The total number of writes
     */
    @Override
    public void onWriteUpdate(int index, int total) {
        Log.e("tetetetete", "sendImageData " + index + " " + total);
    }

    /**
     * Write successfully
     */
    @Override
    public void onWriteSuccess() {

    }

    /**
     * Write failed. 
     * reason has little effect and is only used to troubleshoot the cause of failure
     */
    @Override
    public void onWriteFailed(int i) {

    }
});

# Property description

  1. DeviceInfoFrame is a information frame of the device. In this information frame, screenIdentifier is one of the most important information. Only by matching the brush data obtained through screenIdentifier, can the brush be successful.

    Name Type Description
    firmwareVersion String Firmware version
    batteryLevel String Battery information
    screenIdentifier String Screen information
    macAddress String MAC

  2. TagStatusFrame, Device state information frame

    Name Type Description
    temperature int Device's temperature
    imageId String Device's brush data ID
    timeSinceBoot long The running time after the device is powered on
    rssi int Device's rssi
    deviceStatus int Device operating status indicator
    optionCode int Device-side interactive serial number
    errorCode int Error code (corresponding to serial number)
