# MinewESLv3Kit说明文档

本套SDK仅支持Minew公司出品的蓝牙ESL设备。通过SDK可以帮助开发者处理手机和ESL之间的一切工作，包括：扫描设备，连接设备，向设备写入数据，从设备接收数据等。

MinewESLV3Kit兼容AppCompat或androidx

# 前期工作

整体框架：MTTagBleManager为设备管理类，在APP运行时始终是单例。TagModule是设备实例类，此套件会为每一个设备生成一个TagModule实例，内部包含设备广播数据，该数据会随着设备不停广播而更新。

MTTagBleManager：设备管理类，可以扫描周围的ESL设备，并且可以连接它们，校验它们等；

TagModule：扫描时获取到的设备实例，作为在扫描阶段时期使用；在开始去连接时就能获取到，作为连接设备的实例，读写操作都会用到。

# 开始开发

sdk分为扫描和连接两个阶段。

# 扫描

//获取管理类单例
MTTagBleManager tagBleManager = MTTagBleManager.getInstance();

//在开始扫描前可以设置默认扫描时间，时间毫秒。设置扫描时间不能小于3秒
tagBleManager.setDefaultScanTime(10*1000);

//清除此前的所有扫描结果
tagBleManager.clearScanResult();

//开始扫描并设置监听器，默认扫描60s
tagBleManager.startScan(this, new OnScanTagResultListener() {
    @Override
    public void onScanTagResult(List<TagModule> list) {
		//在这里处理扫描结果
    }

    @Override
    public void onStopScan(List<TagModule> list) {
		//自动停止扫描，返回扫描时间内获取到的所有设备
    }
});

//或使用：开始扫描并设置监听器，设置扫描8秒，设置扫描时间不能小于3秒
tagBleManager.startScan(this,8000， new OnScanTagResultListener() {
    @Override
    public void onScanTagResult(List<TagModule> list) {
		//在这里扫描结果
    }

    @Override
    public void onStopScan(List<TagModule> list) {
		//自动停止扫描，返回扫描时间内获取到的所有设备
    }
});

# 连接

通过扫描方法可获取到周边tag设备，然后可连接指定Tag，连接通过调用connect()。

tagBleManager.connect(tagModule)
1

连接阶段，包括后续写入操作，都需要设置监听器，以判断状态。成功后才能进行刷图，刷图成功后也可以在这里获取到回调通知。

tagBleManager.setOnConnStateListener(new OnConnStateListener() {
    @Override
    public void onUpdateConnState(String s, ConnectionState connectionState) {
        switch (connectionState) {
            case Connecting:
               //开始连接
                break;
            case Connect_Complete:
                //认证成功，此时认为已经连接成功
                break;
            case BRUSH_IMAGE:
               //刷图成功后调用，连接会断开
                break;
            case Disconnect:
                //连接断开
            default:
                break;
        }
    }
});

# 刷图

连接成功后，即可去刷图。首先需要获取设备屏幕信息，通过屏幕信息去获取要写入Tag里面的图像数据。由于蓝牙数据传输长度限制，如果超出限制，将会进行分包处理，然后再依次发送。

//屏幕信息存储在DeviceInfoFrame类中
DeviceInfoFrame frame = (DeviceInfoFrame) tagModule.getMinewFrame(FrameType.DEVICE_INFO_FRAME);
//取出屏幕信息，然后根据屏幕信息去获取刷图数据
int screenIdentifier = frame.getScreenIdentifier();
String macAddress = tagModule.getMacAddress();
//刷图，设备屏幕分双面屏和单面屏，现在是演示刷双面屏，A面图片数据imageABytes，B面图片数据imageBBytes，A面和B面刷图数据不同，设置isSameImage=false,
//如果A面和B面刷图数据相同，设置isSameImage=true,可以提高刷图效率
//如果设备只支持单面牌，默认刷A面屏，设置imageABytes数据，imageBBytes=null，isSameImage=false,
//在刷图场景下，会先执行`onWriteSuccess`再通过`OnConnStateListener`返回`BRUSH_IMAGE`状态；写入失败后断开连接，并通过`onWriteFailed`通知刷图失败并断开连接。
tagBleManager.sendImageData(macAddress, imageABytes,imageBBytes,false, new OnWriteLargeDataListener() {
    
    /**
     * 更新写入进度
     *
     * @param index 当前已写入的数量
     * @param total 总共需要写入的数量
     */
    @Override
    public void onWriteUpdate(int index, int total) {
        Log.e("tetetetete", "sendImageData " + index + " " + total);
    }

    /**
     * 写入成功
     */
    @Override
    public void onWriteSuccess() {

    }

    /**
     * 写入失败。reason作用不大，仅用来排查失败原因
     */
    @Override
    public void onWriteFailed(int i) {

    }
});

# 附表

  1. DeviceInfoFrame，设备信息帧，在该信息帧中，screenIdentifier是最重要的信息之一，通过screenIdentifier获取到相匹配的刷图数据，才能成功刷图。

    名称 类型 描述
    firmwareVersion String 固件版本
    batteryLevel String 电池电量信息
    screenIdentifier int 屏幕尺寸信息
    macAddress String mac

  2. TagStatusFrame，设备状态信息帧

    名称 类型 描述
    temperature int 设备的温度
    imageID String 设备的刷图数据ID
    timeSinceBoot long 设备上电后的运行时间, Units: 秒
    rssi int 设备的rssi
    deviceStatus int 设备运行状态指示位
    optionCode int 设备端交互流水号
    errorCode int 错误码 ( 对应与流水号 )
